# LLM 작업 지시서

docs 폴더의 PROJECT_OVERVIEW.md와 TECH_PLAN.md를 먼저 읽고 이 문서를 본다. 특히 TECH_PLAN의 3.4절(자체 LLM 모델 계획)이 이 작업의 상위 방침이다.

## 1. 이 창이 맡는 것

리라이터의 최종 형태인 자체 LLM을 준비하는 작업 전체. 데이터 만들기, 로컬 모델 돌리기, 나중에 파인튜닝과 평가까지. 작업 위치는 llm 폴더.

역할 분담. 무거운 학습은 나중에 메가존 GPU에서 한다. 이 데스크탑에서는 모델을 돌리는 것(추론)과 데이터 작업까지만 한다.

검토 체계. TECH_PLAN에 정한 대로, 이 파트는 단계마다 구현 전에 별도 검토 창(설계 담당 클로드)의 검토를 거친다. 평가셋 형식, 증류 프롬프트 설계, 파인튜닝 착수 판단, 베이스 모델 선정, 학습 데이터 저작권은 임의로 진행하지 않고 검토된 결정을 받아 구현한다.

## 2. 지금 단계에서 할 일 (두 가지)

### 2.1 로컬 모델 띄우기

목적. 데스크탑에서 한국어 오픈소스 모델을 돌려보고, 백엔드의 리라이터가 외부 API 대신 로컬 모델을 부를 수 있는 선택지를 만든다. 나중에 우리 파인튜닝 모델로 갈아탈 자리를 미리 확인하는 것이기도 하다.

할 일.
- Ollama를 설치하고 한국어를 잘하는 작은 모델을 받아 띄운다. 후보는 EXAONE 계열 3B급을 우선 시도하고, 안 되면 Qwen3 계열 소형. 데스크탑 사양(그래픽카드 메모리)에 맞는 크기를 고른다. 3B급이면 웬만한 데스크탑에서 돈다.
- 백엔드의 리라이터 시스템 프롬프트(BACKEND_SPEC의 prompts 원문)를 그대로 얹어서, 밋밋한 묘사를 넣으면 화면해설풍 설명이 나오는지 손으로 몇 번 시험한다.
- 시각 상태별 규칙(선천맹, 중도실명)을 넣었을 때 색 설명이 실제로 달라지는지 확인한다.
- 결과를 간단히 기록한다. 로컬 소형 모델이 API 대비 품질이 어느 정도인지. 이 기록이 나중에 파인튜닝 필요성 판단의 근거가 된다.

연결. 백엔드의 LLM 호출 함수(call_llm)가 로컬 Ollama를 가리키는 버전도 만들어, 설정으로 API와 로컬을 전환할 수 있게 한다. 이 부분은 backend 폴더의 코드를 건드리므로, 백엔드 창과 겹치지 않게 함수 하나 추가하는 선에서 최소로 한다.

### 2.2 평가셋 첫 삽

목적. 무엇이 좋은 설명인지의 기준을 데이터로 만든다. 이것이 있어야 프롬프트 개선도 파인튜닝도 효과를 잴 수 있다.

할 일.
- llm/data 폴더에 평가셋 파일을 만든다. 형식은 JSON Lines. 한 줄이 한 예시.
- 형식. profile(onset, interest, length), raw(밋밋한 묘사), target(정답 설명). onset은 congenital, acquired, unknown 중 하나.
- 우선 10개에서 20개를 만든다. 미술 작품, 공간, 일상 사물을 섞는다. 같은 raw에 대해 onset만 다른 쌍을 반드시 포함한다(색 설명이 달라지는 것을 재기 위해).
- target 초안은 AI로 생성하되, 원칙(BACKEND_SPEC의 시스템 프롬프트와 시각 상태 규칙)만 주고 생성한다. 방송 대본 등 남의 저작물을 입력하지 않는다.
- 이 평가셋은 학습에 쓰지 않는다. 오직 측정용으로 따로 둔다.

주의. 평가셋의 형식과 예시 방향은 검토 창에서 한 번 검토받은 뒤 수량을 늘린다. 지금은 10개에서 20개 초안까지만.

## 3. 폴더 구조

```
llm/
  data/
    evalset.jsonl      # 평가셋. 측정 전용.
  scripts/             # 증류, 평가 스크립트가 들어갈 자리. 지금은 비워도 됨.
  notes.md             # 시험 결과 기록. 로컬 모델 품질 메모.
```

## 4. 하지 않을 것 (지금 단계)

- 파인튜닝. 데이터가 쌓이고 검토를 거친 뒤 메가존 GPU에서 한다.
- 학습 데이터 대량 생산(증류). 평가셋 형식이 검토를 통과한 다음에 시작한다.
- 방송 화면해설 대본 수집이나 입력. 저작권 규칙 위반. 원칙 참고만 자유.

## 5. 완료 기준 (지금 단계)

- 데스크탑에서 로컬 모델이 돌고, 시스템 프롬프트를 얹어 밋밋한 묘사가 화면해설풍 설명으로 나온다.
- 선천맹과 중도실명 프로필에서 색 설명이 다르게 나오는 것을 확인했다.
- 평가셋 초안 10개에서 20개가 evalset.jsonl에 있고, 같은 raw에 onset만 다른 쌍이 포함되어 있다.
- notes.md에 로컬 모델 품질 소감이 기록되어 있다.
