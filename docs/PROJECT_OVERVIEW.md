# 프로젝트 전체 개요 — 시각장애인 안내 AI

이 문서는 프로젝트 전체의 최상위 설계도다. 클로드 코드는 어떤 부분을 작업하든 이 문서를 먼저 읽고, 여기 정의된 구조와 데이터 형식을 공유 기준으로 삼는다. 세부 작업 문서는 BACKEND_SPEC.md와 WEBSITE_SPEC.md에 있다.

## 1. 우리가 만드는 것

시각장애인용 웨어러블 안내 AI. 안경 카메라로 눈앞의 대상을 보고, 사용자의 시각 상태에 맞춰 생생한 한국어로 설명해준다. 핵심 차별점은 시각 상태(선천맹, 중도실명, 저시력)에 따라 설명, 특히 색 설명을 다르게 한다는 것이다.

## 2. 세 개의 덩어리

프로젝트는 세 덩어리로 이뤄진다. 이 셋은 하나의 프로젝트 폴더 안에 나란히 둔다.

- 백엔드: 실시간으로 이미지를 인식하고 맞춤 설명을 만드는 뇌. 웹사이트와 앱이 공통으로 부른다. 데이터베이스를 관리한다. 작업 문서는 BACKEND_SPEC.md.
- 웹사이트: 회사 소개와 서비스 내용을 담은 공개 페이지, 그리고 작가가 작품 정보를 넣는 입력 페이지. 브라우저에서 본다. 작업 문서는 WEBSITE_SPEC.md.
- 앱: 안경과 연결돼 시각장애인이 쓰는 앱. 폰에서 돈다. 이번 단계에서는 안경 없이 백엔드와 통신하는 뼈대까지만 만든다. 작업 문서는 APP_SPEC.md.

폴더 구조는 이렇게 잡는다.

```
project/
  backend/     # 백엔드. BACKEND_SPEC.md 참고
  website/     # 웹사이트. WEBSITE_SPEC.md 참고
  app/         # 앱. 이번엔 비워둠
  shared/      # 공유 데이터 구조 정의 (아래 4번)
```

## 3. 어떻게 물리는가

가운데 데이터베이스가 있고, 백엔드가 그걸 관리한다.

- 작가가 웹사이트 입력 페이지에서 작품 정보를 넣으면, 백엔드를 통해 데이터베이스에 저장된다.
- 안경 사용자가 앱에서 작품을 비추면, 백엔드가 이미지를 인식하고 데이터베이스에서 그 작품 정보를 찾아, 시각 상태에 맞는 설명을 만들어 돌려준다.

즉 웹사이트는 데이터를 넣는 입구, 앱은 데이터를 꺼내 쓰는 입구이고, 백엔드와 데이터베이스가 공통 심장이다.

## 4. 공유 데이터 구조 (가장 중요)

백엔드와 웹사이트가 반드시 똑같이 봐야 하는 구조다. 이 구조를 shared 폴더에 정의하고 양쪽이 참조한다.

### 작품 (Artwork)
- id: 고유 번호. 시스템이 자동 부여.
- image_url: 작품 사진. 이미지 인식의 기준이자 설명 재료. 필수.
- title: 제목.
- artist: 작가명.
- material: 재질.
- size: 크기.
- year: 제작연도.
- description: 작가가 직접 쓴 설명 글. 선택. 없으면 백엔드가 사진과 기본 정보로 설명을 생성한다.

### 사용자 프로필 (UserProfile)
- user_id: 고유 번호.
- onset: 시각 상태. congenital(선천맹), acquired(중도실명), unknown(불명) 중 하나.
- interest: 관심사. 없어도 됨.
- length: 설명 길이 취향. short, medium, long 중 하나.

이 두 구조는 이번 단계에서 백엔드가 코드 안에 예시 데이터로 갖고 있고, 나중에 실제 데이터베이스로 옮긴다. 웹사이트가 작품을 저장하면 이 Artwork 구조로 저장된다.

## 5. 이번 단계에서 할 것과 안 할 것

할 것:
- 백엔드 뼈대. 이미지를 받아(지금은 인식 가짜) 시각 상태에 맞는 설명을 돌려주는 관 하나.
- 웹사이트 뼈대. 회사 소개와 서비스 페이지, 작가 작품 입력 페이지. 입력한 작품이 백엔드로 저장되는 흐름.
- 앱 뼈대. 안경 없이, 버튼을 누르면 백엔드를 불러 맞춤 설명을 받아 화면에 띄우는 것.
- 세 덩어리가 위 4번 공유 데이터 구조를 똑같이 보게 한다.

안 할 것:
- 실제 안경 SDK, 카메라, 음성 출력. 앱은 백엔드와 통신하는 뼈대까지만. 나머지는 자리만.
- 실제 이미지 인식 모델. 자리만 잡고 가짜로 둔다.
- 실제 데이터베이스. 지금은 코드 안 예시 데이터. 자리만 표시.
- 로그인, 음성 출력, 위치 태깅. 나중 단계. 자리만 남긴다.

## 6. 인식에 대한 방침

작품 식별은 지금 이미지 기준으로 간다. 사용자가 비춘 이미지를 보고 어느 작품인지 맞추고 설명한다. 다만 이미지로 정확히 식별하는 것은 어려운 기술이므로, 나중에 QR이나 NFC 태그로 정확도를 높인다. 이번 뼈대 단계에서는 인식 자체를 가짜로 두고(고정 결과 반환), 실제 인식 모델은 자리만 분리해둔다.

## 7. 공통 규칙

- API 키나 비밀값은 환경변수로만 다룬다. 코드에 하드코딩 금지.
- 각 조각은 함수와 폴더 경계를 분명히 해서, 나중에 진짜로 교체할 때 다른 조각을 안 건드리게 한다.
- 먼저 관이 끝까지 도는 걸 확인한 뒤 조각을 하나씩 진짜로 바꾼다.
- 백엔드와 웹사이트는 4번의 공유 데이터 구조를 어기지 않는다.
